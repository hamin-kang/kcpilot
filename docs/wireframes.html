UX 의사결정
  • 히어로 한 줄로 가치 제안: "5분 만에 사전진단" — 비전문가에게 일상어로 약속한다.
  • CTA 두 개를 나란히: 진단 시작(primary) + 데모 보기(secondary). 망설이는 사용자에게 낮은 부담 경로 제공.
  • "신용카드 불필요" 마이크로카피로 진입 장벽 제거.
  • 3 features는 차별점(속도 / 출처 / 추론 투명성)에 집중. "왜 다른 도구 대신 이걸?"에 답한다.
  • 푸터 면책 문구 상시 노출 — F-04 요구사항: "법적 효력 없음" 명시.

  • LOW 결과는 행 자체에 "전문가 상담 권장" 배지로 사전 경고.
  • 진행 중인 진단도 같은 리스트에 표시 (다른 배경색) — 사용자가 헷갈리지 않게 같은 자리에서 추적.
  • 검색 + 정렬 + 기간 필터. 많은 이력 가진 사용자 대비.
  • 페이지네이션 (무한 스크롤 X) — "내가 봤던 그 진단 위치"가 안정적이어야 함.

UX 의사결정
  • 헤더는 로그인 상태에 따라 두 버전. 로그인 전엔 마케팅 네비, 로그인 후엔 기능 네비.

  • 로고 클릭 시: 로그인 전 → 랜딩, 로그인 후 → 새 진단 페이지로 이동.
  • 푸터의 면책 문구는 진단 결과 화면에선 sticky로 한 번 더 강조 (Screen 5 참조).
  • 고객 문의는 푸터의 "문의하기"로 단일화 (별도 메뉴 없음). MVP는 이메일 폼.
