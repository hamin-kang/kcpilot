# KCpilot 요구사항 정의서

> **프로젝트명:** KCpilot\
> **프로젝트 기간:** 2026.05.15 ~ 2026.07.06 (약 7주)\
> **개발자:** 1인 개발

---

## 1. 프로젝트 개요

### 1.1 프로젝트 정의

중소기업·스타트업이 KC 인증을 받기 위한 사전 진단을 도와주는 AI 기반 웹 서비스. 사용자가 제품 정보를 자연어로 입력하면, AI가 관련 법령·기준을 검색하여 어떤 인증이 필요한지, 어떤 시험 항목을 거쳐야 하는지, 유사 제품 리콜 사례는 무엇인지를 정리해서 보여준다.

### 1.2 해결하려는 문제

중소기업 인증 실태조사(중소기업중앙회) 기준 다음 문제가 입증되어 있다.

- 인증 비용 부담: 44.3%
- 인증 절차 부담: 35.0%
- 인증 취득 기간 부담: 31.6%
- 인증 취득 실패·포기 사유 1위: 까다로운 인증 취득 절차 (29.8%)

기존 정부 시스템 safetykorea.kr은 이미 인증받은 제품을 검색하는 시스템이지, 미인증 제품에 무엇이 필요한지 진단해주는 시스템이 아니다. 사설 컨설팅은 건당 수십~수백만원으로 영세 제조사·스타트업에 부담이다.

### 1.3 타깃 사용자

**1순위 타깃 (Primary):**
- 신제품을 처음 출시하는 스타트업
- 크라우드펀딩(텀블벅, 와디즈)으로 첫 하드웨어 제품 출시하는 1인 창업자
- 해외 제품을 국내에 처음 유통하려는 소규모 수입업자

**2순위 타깃 (Secondary):**
- 기존과 다른 카테고리로 제품을 확장하는 중소제조사 담당자

**비타깃:**
- 동일 품목을 수년째 만들어온 기존 중소제조사 (이미 절차를 안다)
- 인증 담당 직원이 있는 중견·대기업 (자체 처리 가능)

### 1.4 핵심 가치 제안

"제품 정보를 자연어로 입력하면, 어떤 KC 인증이 필요한지 30초 안에 정리된 리포트를 받습니다. 검색에 1~2주 헤매던 일을 30분으로 줄입니다."

### 1.5 차별점

| | safetykorea.kr | 사설 컨설팅 | ChatGPT 검색 | KCpilot |
|---|---|---|---|---|
| 인증 종류 자동 진단 | ❌ | ✅ | △ (부정확) | ✅ |
| 자연어 입력 | ❌ | ✅ | ✅ | ✅ |
| 복수 인증 동시 식별 (안전+전자파+어린이) | ❌ | ✅ | ❌ | ✅ |
| 리콜 사례 사전 경고 | ❌ | △ | ❌ | ✅ |
| 출처 법령 명시 | △ | ✅ | ❌ | ✅ |
| 최신 고시 반영 | ✅ | ✅ | ❌ | ✅ (반자동 갱신) |
| 비용 | 무료 (단 사용 어려움) | 수십~수백만원 | 무료 | 무료~저가 구독 |

가장 결정적인 차별점은 **복수 인증의 동시 식별**이다. 한 제품에 안전인증 + 전자파 인증 + 어린이제품 인증이 동시에 걸리는 케이스를 누락 없이 안내한다. 이게 ChatGPT·검색으로 가장 자주 빠뜨리는 실수다.

## 2. 도메인 정보 (개발자 참고용)

### 2.1 KC 인증의 두 축

KC 인증은 크게 두 가지로 나뉜다.

**축 1 — 안전인증** (산업통상부/KATS 관할)
- 안전인증: 화재·감전 등 위해 우려가 매우 큰 제품 (제품시험 + 공장심사 + 정기검사)
- 안전확인: 위해 우려가 있는 제품 (제품시험만)
- 공급자적합성확인: 위해 우려가 낮은 제품 (자체 시험 또는 제3자 시험 후 신고)

**축 2 — 전자파 인증** (과기부/국립전파연구원 관할)
- 적합인증: 위해 우려 큰 기자재
- 적합등록: 위해 우려 중간
- 자기적합확인: 위해 우려 낮음
- 잠정인증: 기준 미정 기자재

**중요:** 한 제품이 두 축에 동시에 걸릴 수 있다. 예: 헤어드라이어 = 안전인증 + 전자파 인증 둘 다 필요.

### 2.2 제품 카테고리

- 전기용품
- 생활용품
- 어린이제품

한 제품이 복수 카테고리에 걸릴 수 있다. 예: 어린이용 전자완구 = 전기용품 + 어린이제품.

### 2.3 시험기관

KC 인증 시험을 수행하는 기관:
- KTL (한국산업기술시험원) — 종합
- KTR (한국화학융합시험연구원) — 화학·환경 강점
- KTC (한국기계전기전자시험연구원) — 기계·전기 강점
- 그 외 다수 민간 시험기관

전자파 인증은 별도로 국립전파연구원(RRA) 지정 시험기관에서 수행.

KATS는 시험기관이 아니라 제도 관리 기관(정부).

## 3. 기능 요구사항

### 3.1 우선순위 정의

- **Must:** MVP에 반드시 포함. 데모 시연에 필요한 핵심 기능.
- **Should:** 시간 허락하면 포함. 발표 완성도 향상.
- **Could:** 향후 확장 영역.

### 3.2 핵심 기능 (Must)

#### [F-01] 회원가입 및 로그인 (Must)
- **Actor:** 사용자
- **설명:** 이메일/비밀번호 기반 회원가입 및 로그인
- **인증:** JWT 토큰
- **비고:** 진단 이력 관리를 위해 필요

#### [F-02] 제품 정보 입력 (Must)
- **Actor:** 로그인한 사용자
- **입력:**
  - 제품명 (필수)
  - 제품 용도 (필수)
  - 전기 사용 여부 (필수)
  - 어린이 대상 여부 (필수)
  - 주요 사양 (자유 텍스트, 선택)
  - 사양서 PDF (선택, 업로드)
- **출력:** 진단 ID 생성, 다음 단계 진행
- **예외:** 필수 필드 누락 시 추가 입력 요청

#### [F-03] AI 기반 인증 진단 (Must)
- **Actor:** 시스템
- **설명:** 입력 정보를 바탕으로 LangGraph 멀티에이전트가 적용 인증 진단
- **처리 단계:**
  1. 카테고리 분류 (전기/생활/어린이)
  2. 적용 인증 식별 (안전인증/안전확인/공급자적합성확인/전자파 인증 등)
  3. 시험 항목 추출
  4. 시험기관 추천
  5. 유사 리콜 사례 검색
  6. 자기비판 노드를 통한 결과 검증
- **출력:** 진단 결과 JSON
- **응답 시간 목표:** 30~60초 이내

#### [F-04] 진단 결과 표시 (Must)
- **Actor:** 사용자
- **표시 항목:**
  - 적용 인증 종류 (복수 가능)
  - 적용 카테고리 (전기/생활/어린이 중 해당)
  - 위해 단계 (안전인증/안전확인/공급자적합성확인)
  - 적용 시험 항목 목록
  - 시험기관 추천 (KTL·KTR·KTC 비교)
  - 유사 리콜 사례 경고
  - 신뢰도 표시 (High/Medium/Low)
  - 출처 법령 명시
  - **면책 문구 고정 표시**

#### [F-05] 모호 케이스 처리 (Must)
- **Actor:** 시스템
- **설명:** 신뢰도가 낮거나 경계선 케이스인 경우, AI가 단정하지 않고 솔직히 모호함을 표시
- **응답 예시:** "이 제품은 통제번호 X의 일부 기준에 근접합니다. 다만 ○○ 조항과 중첩되어 확신하기 어렵습니다. 한국제품안전관리원 또는 KTL에 직접 문의를 권장합니다."
- **목적:** AI가 거짓 자신감으로 사용자를 오도하지 않도록 함

#### [F-06] 단계별 추론 로그 표시 (Must)
- **Actor:** 사용자
- **설명:** AI가 어떤 단계를 거쳐 결론에 도달했는지 펼쳐서 볼 수 있음
- **목적:** 설명가능성(Explainable AI). 사용자 신뢰도 확보. 발표 시 핵심 시연 포인트.

#### [F-07] 출처 법령 표시 (Must)
- **Actor:** 시스템
- **설명:** 모든 진단 결과에 RAG가 참조한 법령 조항 명시
- **표시 형식:** "근거: 전기용품 및 생활용품 안전관리법 시행규칙 별표 3 제○호"

#### [F-08] 진단 이력 조회 (Must)
- **Actor:** 사용자
- **설명:** 본인이 진행한 진단 이력 목록 조회 및 상세 보기
- **정렬:** 최신순

### 3.3 보조 기능 (Should)

#### [F-09] 사양서 PDF OCR 추출 (Should)
- **설명:** 사용자가 사양서 PDF 업로드 시 OCR로 사양 자동 추출
- **목적:** 입력 부담 감소
- **기술:** pdfplumber + LLM 후처리

#### [F-10] 표시 의무 체크리스트 (Should)
- **설명:** KC 마크, 모델명, 정격, 제조업체, A/S 연락처 등 의무 표시 항목 자동 생성
- **목적:** 표시 의무 누락 방지

#### [F-11] 시험 의뢰서 초안 자동 생성 (Should)
- **설명:** 진단 결과를 바탕으로 KTL 시험 의뢰서 초안을 PDF로 자동 생성
- **목적:** 신청서 작성 부담 감소

### 3.4 확장 기능 (Could)

#### [F-12] 공급자적합성확인서 자동 작성 (Could)
- **설명:** 공급자적합성확인 대상 제품의 경우, 확인서 양식 자동 작성
- **목적:** 자체 신고 서류 작성 부담 감소

#### [F-13] 데이터 갱신 알림 (Could)
- **설명:** 등록한 제품 카테고리에 신규 리콜 사례 또는 고시 개정이 있으면 사용자에게 알림
- **목적:** 사후 모니터링

## 4. 비기능 요구사항

### 4.1 성능

- 진단 결과 응답 시간: 60초 이내 (목표 30초)
- 동시 사용자: 데모 환경 기준 10명 수준
- DB 응답 시간: 단순 조회 1초 이내

### 4.2 신뢰성 (가장 중요)

- **AI 거짓 자신감 방지:** 모든 답변은 RAG로 검색된 법령 텍스트에 근거해야 함. 법령에 없는 내용 생성 금지.
- **출처 표시 강제:** 모든 진단 결과에 출처 법령 조항 명시.
- **신뢰도 표시:** RAG 결과 명확도에 따라 High/Medium/Low 신뢰도 표시.
- **자기비판 노드:** LangGraph에 자기비판 노드 포함하여 1차 결론 재검토.
- **면책 문구 상시 표시:** 모든 결과 화면 하단에 "본 서비스는 사전 참고용이며 법적 효력이 없습니다" 고정 표시.

### 4.3 사용성

- 비전문가(중소기업 사장)가 이해할 수 있는 수준의 설명
- 모바일/태블릿/데스크톱 반응형 지원
- 한국어 우선

### 4.4 보안

- 사용자 비밀번호 BCrypt 해싱
- JWT 토큰 만료 시간 설정
- API 요청 Rate Limiting (악용 방지)
- HTTPS 적용 (배포 시)

### 4.5 데이터 관리

- 법령 데이터: 반자동 갱신 (관리자 수동 트리거 + 정기 점검)
- 리콜 DB: safetykorea Open API로 일 1회 자동 갱신
- KC 인증 정보: safetykorea Open API로 일 1회 자동 갱신
- 모든 답변에 "정보 기준일" 명시

### 4.6 확장성

- 백엔드와 AI 서비스를 별도 마이크로서비스로 분리
- 향후 다른 인증(CE, FCC 등)으로 확장 가능한 구조

- ORM: JPA (Hibernate)
- DB: PostgreSQL 18 (pgvector 0.8.2 확장 포함)
- API 문서: springdoc-openapi 2.8.17 (Swagger UI 자동 생성)
- 인증: Spring Security + JWT
- 배포: Docker

### 5.2 AI 서비스

- 언어: Python 3.11.15
- 프레임워크: FastAPI 0.115.12 + Uvicorn 0.34.3
- AI 프레임워크: LangChain 1.3.1 + LangGraph 1.2.0

- LLM: OpenAI GPT-4o 또는 Anthropic Claude API

### 5.3 프론트엔드

- 프레임워크: Next.js 15.5.18 (App Router) + React 19.2.6

- 스타일링: Tailwind CSS 3.4.1
- 테스트: Jest 30.4.2 + @testing-library/react 16.3.2
- 비고: Claude Code 적극 활용. 솔로 개발자가 직접 작성하기보다 AI 보조 코딩 위주.

### 5.4 인프라

- DB: PostgreSQL 18 + pgvector 0.8.2 (Docker, 이미지 `pgvector/pgvector:0.8.2-pg18`)

- 개발 환경: Ubuntu

## 6. 데이터 요구사항

### 6.1 RDBMS에 저장할 데이터

- 사용자 정보
- 제품 정보
- 진단 이력
- 안전관리대상 품목 마스터 (약 241개 품목, MVP에서는 20~30개)
- 시험기관 마스터
- 리콜 사례 (safetykorea API 동기화)
- KC 인증 정보 (safetykorea API 동기화)

### 6.2 VectorDB(pgvector)에 저장할 데이터

- 전기용품 및 생활용품 안전관리법 본문
- 어린이제품 안전특별법 본문
- 시행규칙 별표 (품목 목록)
- 품목별 안전기준 (MVP에서는 20~30개 품목 우선)
- 전자파 인증 관련 기준
- safetykorea FAQ
- 리콜 사례 (검색용)

### 6.3 데이터 갱신 정책

- 리콜 DB: 일 1회 자동 갱신 (Spring `@Scheduled`)
- KC 인증 정보: 일 1회 자동 갱신
- 법령 본문: 반자동 (국가법령정보센터 변경 감지 → 관리자에게 알림)
- 안전기준 고시: 수동 갱신 (개정 시)

## 9. MVP 범위 (7주 안에 반드시 완성할 것)

### 9.1 MVP에 포함되는 것

- 회원가입/로그인
- 제품 정보 입력 (텍스트 기반)

- 시나리오 A (헤어드라이어 케이스) 안정적 시연
- 시나리오 B (어린이 전자완구 이중 카테고리) 시연
- 시나리오 C (모호 케이스, 전문 상담 유도) 시연

### 9.3 시연 시나리오

**시나리오 A — 헤어드라이어 (안전인증 + 전자파 인증 동시)**
- 입력: 제품명 "헤어드라이어", 220V 1200W, 가정용
- 출력: 안전인증 대상 + 전자파 적합등록 동시 필요, 시험 항목 12개, KTL 추천, 유사 리콜 사례 3건

**시나리오 B — 어린이 전자완구 (이중 카테고리)**
- 입력: 제품명 "스마트 학습 인형", 만 5세 대상, 충전식 배터리
- 출력: 어린이제품 + 전기용품 동시 인증 필요, 환경호르몬 시험 + 전기 안전확인 시험, 작은 부품 질식 위험 경고

**시나리오 C — 산업용 IoT 센서 (모호 케이스)**
- 입력: 제품명 "공장용 환경 센서", 12V DC, B2B 전용
- 출력: 신뢰도 낮음 표시, 산업용은 면제 가능성 있으나 사용 환경에 따라 다름, 한국제품안전관리원 직접 문의 안내
