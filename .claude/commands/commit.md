다음 단계를 순서대로 실행해줘.

**1단계: 스테이징된 변경사항 확인**

`git diff --staged` 를 실행해서 스테이징된 변경사항을 파악해줘.
변경사항이 없으면 "스테이징된 변경사항이 없습니다. `git add` 로 파일을 먼저 스테이징해주세요." 라고 알리고 종료해줘.

**2단계: 커밋 메시지 작성**

아래 kcpilot 프로젝트 커밋 컨벤션에 맞게 커밋 메시지를 작성해줘.

**형식:** `type: subject`

**타입(type) 선택 기준:**
| 타입 | 설명 |
|------|------|
| feat | 새로운 기능 추가 |
| fix | 버그 수정 |
| docs | 문서 수정 (README 등) |
| style | 코드 포맷팅, 세미콜론 누락 수정 (로직 변경 없음) |
| refactor | 코드 리팩토링 |
| test | 테스트 코드 추가 및 수정 |
| chore | 빌드 업무, 패키지 매니저 설정 수정 |

**subject 작성 규칙:**
- 마침표(`.`) 사용 금지
- 과거형이 아닌 현재형/명령조 사용 (예: 수정했음 X → 수정 O)
- 변경사항을 간결하고 명확하게 요약

작성한 커밋 메시지를 코드 블록으로 보여줘.

**3단계: 커밋 실행 여부 확인**

"위 메시지로 커밋할까요?" 라고 물어봐줘.
사용자가 승인하면 `git commit -m "커밋메시지"` 를 실행해줘.
